How it Works (Pitch Deck)

This interactive mockup shows how the client-side sheet parser collaborates with the ESP32 to drive piano lights and DSP visualization.

1

PDF Input & Rendering

A user drops a sheet music PDF. The app uses PDF.js to render pages to canvases.

2

Gemini AI Multimodal OMR

We convert the canvas to images, and send them to the Gemini API with a JSON schema constraint.

3

Unified Event JSON & MIDI

The parser outputs structured piano notes: [{time: 1.2, note: "C4", dur: 0.5}].

4

ESP32 Light sync & DSP

Notes are streamed to ESP32 via WebSocket (MIDI bytes) to light up LEDs. DSP listens to verify if the correct note was played!

Key Selling Points for Judges:

  • AI + Edge Collaboration: Heavy visual computation (OMR) runs in-browser using Gemini API; Edge (ESP32) handles low-latency note lights & audio DSP.
  • 100% Client-side: No custom backend server to crash. Runs straight off local file or ESP32 SPIFFS.
  • Offline Capability: Instantly parses MIDI/MusicXML files locally if internet fails.
